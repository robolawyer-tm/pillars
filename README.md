# Pillars

Foundational architectural ideas, design philosophies, and forward-thinking proposals for the robolawyer-tm ecosystem.

These are the **core concepts** that inform how star-bridge and secret-server (and future projects) are built, organized, and understood.

## Contents

- **[DESIGN_PHILOSOPHY.md](DESIGN_PHILOSOPHY.md)** — How process and structure interrelate; the case for dynamic notation
- **[NOTATION_VISION.md](NOTATION_VISION.md)** — Proposal for accessible, visual representation of installation/process flows
- **[MESH_TOPOLOGY_VISION.md](MESH_TOPOLOGY_VISION.md)** — Phone-to-phone bridging; peer discovery and SSH mesh networking
- More to come...

## Purpose

Rather than bury important architectural decisions deep in implementation code, pillars documents them at the thought level—making them retrievable, linker-friendly, and extensible.

These are ideas that should inform not just the current projects, but become reference points as new projects emerge.
